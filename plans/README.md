## Roadmap backlog (audited/agreed, not yet turned into plans)

Ask the advisor to write any of these as the next numbered plan.

### Product slices toward the vision

- **Import → profile**: upload an existing resume (PDF/DOCX) or paste raw text →
  extract text → AI parses into the profile schema → save as a profile. Grounds:
  profiles are manual-entry only today; `react-dropzone` + `file-uploader.tsx` +
  Cloudinary already in the stack; a commented-out "Import an existing resume
  (LinkedIn, JSON Resume)" card exists in `src/app/dashboard/resume/page.tsx:96-121`.
  Effort: L.
- **ATS score + keyword report**: rank a generated resume against the JD —
  extract required keywords, coverage %, missing-keyword gaps, and an AI "why it
  scores X" summary. This is the core differentiator of "best ATS generator" and
  does not exist yet. Effort: L.
- **Generation quality**: make AI output trustworthy — replace the shared global
  Gemini chat session (cross-user data-bleed bug, see audit finding C1 below)
  with stateless per-request generation, add a stronger ATS-keyword prompt and
  structured output. Effort: M. Foundation for the two slices above.

### Production hardening (from the audit — required for "production level")

- **[S1 → now plan 004] IDOR / broken object-level auth (P1, HIGH):** `getResume`,
  `updateResume`, `getProfileResumes`, `uploadPreviewImage`
  (`src/server/routers/resume-router.ts`), `updateProfile`
  (`profile-router.ts`), the `profileId` in `createResume`, and the RSC
  `src/app/dashboard/resume/edit/[id]/page.tsx` all fetch/mutate by ID with **no
  ownership check** — any signed-in user can read/overwrite anyone's resumes and
  profiles. `src/app/dashboard/resume/page.tsx:43-64` already shows the correct
  ownership pattern to copy. Effort: M.
- **[S2 → now plan 003] Auth bypass + swallowed 401 (P1, HIGH):** `src/server/jstack.ts` accepts
  `Authorization: Bearer <account_id>` as valid auth (account IDs are non-secret
  nanoids returned to clients), and its `try/catch` swallows the 401 instead of
  re-throwing. Effort: S.
- **[C1 → now plan 005] Shared Gemini session (P1, HIGH):** `src/server/services/google-ai-model.ts:18`
  exports one module-level `startChat` shared by all users; history accumulates
  across users (PII bleed), grows unbounded, and races under concurrency.
  Effort: S–M.
- **[DX] Verification baseline (P1):** no `typecheck` script, no tests, no test
  runner. Add `tsc --noEmit` + Vitest + a router-test harness so the security
  fixes get a regression net. Effort: S+M.
- **[S4] Sensitive-data logging (P2):** ~20 `console.log` sites dump user/account
  objects, the bearer token (`jstack.ts:19`), raw AI responses
  (`ai-resume.ts:116`), and resume PII on every render. Effort: S. (002 removes
  the two hottest client-side ones.)
- **[C2] Template empty-section guards emit `0` → PDF crash/blank preview:**
  `templateOne/Two` use `{arr.length && …}`; `templateThree/Four` use
  `length ?? 0 > 0` (precedence bug). Effort: S.
- **[T2/T3] Dead starter scaffolding + ~13 unused deps:** `features/products`,
  `features/kanban`, `features/overview`, `constants/mock-api.ts`, `post-router.ts`
  + `posts` table, duplicate theme components, `types/next-auth.d.ts`,
  `wrangler.toml`, and unused deps (`better-auth`, `crypto-js`, `crypto-browserify`,
  `stream-browserify`, `buffer`, `sort-by`, …). Effort: M.
- **[T5] Templates duplicate rendering logic:** the 4 `@react-pdf` templates
  re-implement the same section rendering; extract shared primitives. Effort: L
  (do after a verification baseline exists).
